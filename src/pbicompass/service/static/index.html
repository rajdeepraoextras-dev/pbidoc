Questions, answered

Before you upload.

Is my data uploaded or stored?

No. PBICompass extracts semantic-model metadata only — tables, columns, measures, relationships, DAX, RLS roles — never row-level data. Your file is processed in an isolated sandbox that's shredded the moment your documents are rendered.

Can it run fully offline?

Yes. The deterministic engine produces complete documentation with no API key and no external calls. AI providers are optional, for richer prose only.

Is .pbix supported?

Yes, including legacy binary exports — metadata is read directly from the compressed data model, never the row-level tables.

Is .pbip supported?

Yes — both the modern TMDL format and legacy TMSL (model.bim), plus PBIR report layouts, enhanced and legacy.

Does it work with Microsoft Fabric?

Yes — Fabric semantic models export to the same .pbip/TMDL structure PBICompass already parses.

What about large, enterprise-scale models?

Uploads up to 100MB are supported out of the box (configurable for self-hosted deployments) — built for real models with hundreds of measures, not just samples.

What export formats are available?

Markdown, HTML, DOCX, PDF and JSON — every document, every format, from one parse.

Which AI models power the documentation?

You choose: Claude, Gemini, Cohere, or 1000+ models via MeshAPI — bring your own key, used only for that job, never stored. Or skip AI entirely with the deterministic offline engine.
